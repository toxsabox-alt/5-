# Сайт — Оксана Терпенёва

Статический лендинг (HTML, CSS, JS), без сборщика.

## Инструкция по развёртыванию

### 1. Состав проекта

В корне репозитория должны лежать `index.html`, `contact.html`, папки `css/`, `js/`, `assets/`, `services/`, `blog/`, `about/`, `calculator/`, `privacy/`, `consent/` — со своими `index.html`, где нужно. Не переименовывайте пути без правки ссылок в файлах.

### 2. Локальный просмотр

- **VS Code / Cursor:** расширение «Live Server» → правый клик по `index.html` → Open with Live Server.
- **Терминал (Python):** в папке проекта выполните `python -m http.server 8080` и откройте в браузере `http://localhost:8080`.

Так корректно отрабатывают относительные пути к CSS и скриптам; при открытии файла напрямую с диска (`file://`) часть функций может вести себя иначе.

### 3. GitHub Pages

1. Создайте репозиторий и загрузите в него все файлы проекта.
2. В настройках репозитория: **Pages** → Source: **Deploy from a branch** → ветка `main`, папка `/ (root)` → Save.
3. Сайт будет доступен по адресу вида `https://<user>.github.io/<repo>/`. Если репозиторий называется `username.github.io`, корень сайта — без дополнительного сегмента в пути.

### 4. Netlify (перетаскивание)

1. Зайдите на [netlify.com](https://www.netlify.com/).
2. Перетащите в область deploy всю папку проекта (нужна папка с `index.html` в корне, а не ZIP внутри другой папки без `index.html`).
3. Присвойте имя сайта в настройках домена.

### 5. Обычный хостинг (FTP / файловый менеджер)

1. В панели хостинга найдите каталог сайта (часто `public_html` или `www`).
2. Загрузите **содержимое** проекта так, чтобы `index.html` оказался в корне сайта.
3. Убедитесь, что загрузились и подпапки (`css`, `js`, `blog` и т.д.).

### 6. Контакты и тема оформления

Почта и Telegram задаются в файле `js/config.js` (поля `email` и `telegram`). После правки обновите кэш браузера (Ctrl+F5).

### 7. Проверка после публикации

- Откройте главную, страницу «Услуги», одну статью блога, форму контактов.
- Проверьте переключение светлой/тёмной темы и мобильную вёрстку.
- Протестируйте кнопку отправки формы (открытие почтового клиента).
